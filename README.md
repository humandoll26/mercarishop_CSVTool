メルカリShops向けの下書き商品CSVを、ローカルで複数件まとめて作るためのシンプルなブラウザツールです。

このツールは、メルカリShopsの商品情報を一括登録するための「下書きCSV」を作る用途を想定しています。CSV作成後は、メルカリShops管理画面のCSV一括登録から読み込み、作成された下書きページ側で画像を登録していく運用を前提にしています。

## できること

- 複数商品レコードの作成、複製、削除
- カテゴリCSVを使った階層選択
- ブランドCSVを使った検索選択
- CSVの事前検証
- S-JIS / UTF-8 BOM付き でのCSVダウンロード
- 入力中データ、既定値、文字コード設定、読込済みマスターの `localStorage` 保存

## ファイル構成

- `index.html`
- `script.js`
- `style.css`
- `encoding-japanese.min.js`
- `brand_master_sjis.csv`
- `category_master_updated_sjis.csv`

## 使い方

1. `index.html` をブラウザで開きます。
2. 商品情報を入力します。
3. カテゴリは検索または階層選択で設定します。
4. 必要ならブランドCSVを読み込んでブランドを設定します。
5. `全件チェック` でエラーを確認します。
6. `CSVダウンロード` で出力します。

## 想定ワークフロー

1. このツールで、商品説明・価格・在庫など画像以外の商品情報を入力します。
2. 出力したCSVを、メルカリShops管理画面のCSV一括登録から読み込みます。
3. CSV読込後に作成された下書き商品ページを開きます。
4. 各下書きページで商品画像を登録します。
5. 最終確認をして公開します。

## このツールで扱うもの / 扱わないもの

- 扱うもの: 商品名、商品説明、価格、在庫、カテゴリ、ブランド、発送設定など、CSVでまとめて下書き登録しやすい項目
- 扱うもの: SKU情報。SKUは1つの商品レコードに対するバリエーション項目で、色やサイズ違いなどを表すためのもの。このツールは「1商品レコードに1つのパターンの商品を紐づける」前提で作っています
- 扱わないもの: 商品画像の登録

画像はCSVに含めて一括投入するのではなく、CSV読込後に作成されたメルカリShops側の下書きページで順番に登録していく想定です。

## 保存される内容

このツールはブラウザの `localStorage` を使います。以下を自動保存します。

- 商品レコード
- 一部選択項目の既定値
- 文字コード設定
- 読み込んだブランドマスター
- 読み込んだカテゴリマスター

ブラウザの保存領域を消すと、これらの内容も消えます。

## 注意

- メルカリShopsのCSV仕様変更には追従していません。利用前に最新仕様と突き合わせてください。
- このツールはCSV下書きの作成補助用です。実際の登録はメルカリShops管理画面のCSV一括登録で行ってください。
- 商品画像はこのツールでは登録しません。CSV読込後に生成された下書きページで登録してください。
- `商品画像名_1` など、CSV上は存在するがV1ではUI入力を持たない項目があります。
- カテゴリ・ブランドマスターはブラウザ保存容量の都合で、環境によっては保存に失敗する可能性があります。
- ローカルHTMLを `file://` で直接開いた場合、ブラウザによっては同梱CSVの自動読込が不安定です。その場合は `開く` ボタンから手動読込してください。
- `localStorage` はブラウザ単位の保存です。別ブラウザやシークレットウィンドウには引き継がれません。

## V1で割り切っていること

- SKUは1つの商品に対する色・サイズなどのバリエーション管理に使う項目です
- このツールは、1商品レコードに1つのパターンの商品を紐づける前提です
- V1ではSKU入力は実質 `SKU1` 中心です
- 商品画像名の複数入力UIは未対応です
- 警告表示は出さず、エラー表示のみです
- 外部API連携、認証、クラウド保存はありません
- 高度な一括編集機能はありません
